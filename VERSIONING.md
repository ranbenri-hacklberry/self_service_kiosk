# מדיניות גרסאות (Versioning Policy)

## מבנה הגרסה: `X.Y.Z`

| חלק | שם | מתי משתנה | דוגמה |
|------|------|-----------|-------|
| **X** | גרסה ראשית (Major) | שינוי גדול / פיצ'ר חדש משמעותי | הוספת מערכת משלוחים, הוספת AI |
| **Y** | תת-גרסה (Minor) | שינוי פיצ'ר קיים / שיפור משמעותי | עיגול חכם במלאי, שינוי לוגיקת תצוגה |
| **Z** | תיקון באג (Patch) | תיקון באג / שינוי קטן | תיקון תצוגה, תיקון שגיאת SQL |

## דוגמאות

- `0.9.0` → `1.0.0` = הוספת מערכת משלוחים חדשה (Major)
- `0.9.0` → `0.10.0` = הוספת עיגול חכם לספירת מלאי (Minor)  
- `0.9.0` → `0.9.1` = תיקון באג בתצוגת מחיר (Patch)

## איפה מופיעה הגרסה

1. **מסך כניסה** - מתחת לכפתור יציאה
2. **בר עליון** - בקטן בצד שמאל (בכל המסכים חוץ ממנהל)
3. **Maya Dashboard** - בנתוני המערכת למעלה

## קובץ הגרסה

`src/config/version.js`

```javascript
export const APP_VERSION = 'β X.Y.Z';
export const VERSION_DATE = 'YYYY-MM-DD';
export const VERSION_NOTES = 'תיאור קצר של השינוי';
```

## לפני העלאה ל-main

1. עדכן את `version.js` בהתאם לסוג השינוי
2. ודא ש-VERSION_DATE מעודכן
3. הוסף VERSION_NOTES תמציתי
